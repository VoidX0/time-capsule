## 关于贡献
- 🤝 欢迎任何形式的贡献，包括代码、文档、测试和建议
- 🌟 贡献有助于改进本项目，也让更多用户受益
- 📝 在贡献之前，请先阅读本指南，了解项目流程和规范

## 贡献代码
- 🍴 Fork 本项目到你的 GitHub 账号
- 💻 克隆到本地开发环境
```bash
git clone https://github.com/VoidX0/time-capsule.git
cd time-capsule
```
- 🔧 创建新的分支进行开发（遵循 feature/xxx 或 bugfix/xxx 命名）
```bash
git checkout -b feature/add-new-feature
```
- 🧪 完成功能或修复后，提交代码并推送到远程分支
- 📄 提交 Pull Request，描述修改内容和目的

## 文档贡献
- 📝 可以贡献新的文档、修复错别字或优化示例
- 📂 文档文件位于 `docs/content` 目录
- 🔗 提交 Pull Request 时，请保持文档结构清晰，并检查链接是否有效

## 提问与反馈
- ❓ 遇到问题或发现 Bug，请先查看 Issues
- 🐛 提交新的 Issue 时，请提供尽量详细的信息，包括：
```
环境信息（操作系统、浏览器/客户端版本）
重现步骤
截图或日志（如有）
💬 对功能或体验有建议，也欢迎在 Issues 中提出
```

## 代码规范
- 🖋️ 使用 TypeScript/JavaScript、C# 代码风格请参考现有项目规范
- 🧹 保持代码整洁，变量命名清晰
- 📦 提交前确保项目可以成功构建并通过测试
- 🔄 在 Pull Request 前请先同步主分支，避免冲突

## 社区和沟通
- 🌐 可加入项目讨论或使用 GitHub Discussions
- 📢 欢迎在 Issues 或 Discussions 中分享使用经验或最佳实践
